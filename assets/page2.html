Fix for Android WebView Issue 17535

This sample app demonstrates a fix for Android webview issue 17535: links that point to valid HTML assets but contain parameters or anchors will cause the webview to incorrectly display a page not found error. This serious issue affects Android 3.0, 3.1, 3.2 and 4.0 and is finally fixed in Android 4.1.

How to test

Use the links below to confirm that the fixed webview works correctly on affected OS versions. You should see no 404 errors when using the anchor and param links.

You are currently on Page2.

Load Page1 with anchors:
file:///android_asset/page1.html#test

Load Page1 with params:
file:///android_asset/page1.html?test=test
